# plan-schedule

# 【要件定義書】オフィス管理者向けビル運営DX Webアプリケーション（仮）

## 1. プロダクト概要
本プロダクトは、オフィスビルのプロパティマネジメント（PM）業務における「サイネージのコンテンツ不足」「お知らせ作成・マルチポストの業務負荷」「スケジュールの視認性の悪さ」を、AIとデータベースの一元化によって解決するWebアプリケーションである。

管理者が1度情報を入力するだけで、AIが各種メディア（サイネージ、メール、ポータル、PDF）に最適化されたコンテンツを自動生成・配信し、同時に館内スケジュールを可視化する。

---

## 2. 対象ユーザーとペイン
* **対象ユーザー:** オフィスビルの管理会社、PM（プロパティマネジメント）担当者、ビルマネジメント会社
* **抱えるペイン:**
  1. サイネージの有料コンテンツ（ニュース・天気等）を契約しているが、同じ内容がループするだけで費用対効果が低い。独自のコンテンツを作る余裕がない。
  2. 工事や点検のお知らせを、サイネージ（画像）、メール、入居者ポータル、掲示板（PDF）など、媒体ごとに手作業で個別作成・加工しており、業務負荷が非常に重い。
  3. 各種イベントや工事が「点のお知らせ」になっており、月間・年間のスケジュールとして視覚的に俯瞰できない。

---

## 3. システム機能要件

### 3.1. コア機能一覧（追記・アップデート分）

#### 柱①：AIサイネージコンテンツ自動生成機能
* [ ] **通常時・イベント報告映像の自動配信（新規）:**
  * ビル内で行われたイベントの実施後、その様子を伝える紹介テキストや映像をサイネージ用に自動編成する機能。
* [ ] **プライバシー保護映像加工（AI匿名化）（新規）:**
  * 館内カメラやイベント時の撮影映像をサイネージに流す際、AIが映像内の「人物の顔」や「特定個人の情報」を自動で検知。
  * ぼかし、モザイク、またはアバター化等の**「人が見えない（個人を特定できない）加工」**をリアルタイム、もしくは半自動で行い、プライバシーに配慮した状態でビルの活気（賑わい）を映像コンテンツとして配信できること。

#### 柱②：ワンソース・マルチアウトプット機能（AIお知らせ生成・配信）
* [ ] **簡易情報入力フォーマット:** 管理者が「工事・点検内容」をフリーテキスト（一言）または所定の定型フォーマットに入力できる画面。
* [ ] **マルチメディア一括生成:** 1度の入力データから、AIが裏側で以下の媒体用コンテンツを**同時に一括生成**すること。
  * [ ] サイネージ用案内画像
  * [ ] テナント向け通知メール本文（テキスト）
  * [ ] 入居者ポータルサイト用掲載テキスト
  * [ ] 館内掲示板用案内（PDF出力用レイアウト）
* [ ] **ワンクリック配信確認（エージェント機能）:** 生成された各メディア向けのコンテンツを管理者が確認し、「このまま配信（出力）する」ボタンを押すだけで、対象メディアへの自動投稿やメール送信、PDFダウンロードが実行されること。

#### 柱③：タイムライン・カレンダー自動可視化機能
* [ ] **カレンダー自動マッピング:** 「柱②」で入力・確定された工事や点検、イベントのスケジュール情報が、共通データベースを通じてカレンダー画面（月間・週間ビュー）へ自動でプロットされること。
* [ ] **ダッシュボード視認性:** 管理者および（将来的には）入居者が、ビルで「いつ・何が起きるか」を一目で俯瞰できるタイムライン・カレンダー画面を有すること。

#### 柱④：防災・緊急時モード（BCP対策機能）
* [ ] **緊急ワンタップ切り替え:** 地震、台風、火災、近隣の鉄道遅延などの緊急事態が発生した際、管理者が管理画面からワンタップ（または一言入力）するだけで、システム全体を「防災モード」に切り替えられること。
* [ ] **サイネージの緊急画面ジャック:** 配信中の通常コンテンツを強制的に中断し、避難経路、防災情報、近隣のインフラ状況（運行情報等）を表示する緊急画面に切り替えること。
* [ ] **テナント向け緊急一斉連絡:** サイネージの切り替えと同時に、登録されているテナント担当者宛てに、状況説明やビル側の対応方針（例：「ロビーを開放します」等）を記載した緊急メールをAIが自動作成・一斉送信すること。

## 4. 画面構成案（モックアップ要件）
GitHubのIssue等で画面イメージを議論するための基本構成案。

1. **ダッシュボード（ホーム）**
   * 今月・今週のビルスケジュール（カレンダー表示）
   * 現在サイネージに配信中のコンテンツプレビュー
2. **お知らせ作成・配信画面**
   * 点検・工事情報の入力フォーム
   * AI生成結果のプレビュー領域（サイネージ画像 / メール文 / ポータル文 / PDFイメージがタブで切り替えられる構成）
   * 「一斉配信・確定」ボタン
3. **AIサイネージ自動配信 設定画面**
   * 配信スケジュール設定（毎日、毎週、時間指定）
   * 生成コンテンツのテーマ・プロンプト管理（例：「近くのおすすめランチ情報」「季節の挨拶と防災の知恵」など）

## 5. 技術スタック・アーキテクチャ選定の視点（開発向けメモ）
* **フロントエンド/バックエンド:** 迅速なプロトタイプ開発が可能な構成（例：Next.js, FastAPI等）
* **AI・LLM連携:** * 画像生成：DALL-E 3, Midjourney API, もしくは テンプレートベースの画像合成エンジン
  * テキスト生成・リライト：OpenAI API (GPT-4o) / Anthropic Claude API等
* **インフラ・データベース:** スケジュール管理とお知らせデータを一元管理するリレーショナルデータベース（PostgreSQL等）
